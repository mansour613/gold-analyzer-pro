# Gold Analyzer Pro V11.18 — Backend Signal Engine + Daily Range Fix

## Implemented

### Backend Signal Engine
- Signals are generated by the Render backend through `/api/gold/signal?interval=...`.
- Multi-timeframe scan is generated through `/api/gold/signals/scan`.
- The mobile frontend no longer uses the old local `signalEngine.ts`; it displays backend-calibrated signal output.
- Signal history continues to store/display the backend signal results.

### Chart/Home Daily High & Low
- `/api/gold/quote` now enriches the quote with the latest completed **daily candle** high/low.
- When the market is closed, Day High and Day Low come from the last completed 1D candle instead of stale/partial intraday candles.
- Added response fields:
  - `dailyRangeSource`
  - `dailyCandleTime`

## Expected Behavior
- Chart page signal strip uses backend signal data.
- Chart page Day High/Low matches the last real daily candle during market-closed periods.
- Home Day High/Low also benefits from the corrected quote endpoint.

## Deployment Notes
- Render must have:
  - `TWELVE_DATA_API_KEY`
  - optional `GOLDAPI_KEY`
  - `OPENROUTER_API_KEY`
- Vercel must have:
  - `VITE_API_BASE_URL=https://gold-analyzer-pro.onrender.com`
